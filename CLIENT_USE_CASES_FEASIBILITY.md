## Use Case 1: ‚úÖ –û—Ñ–æ—Ä–º–∏—Ç—å –∑–∞–∫–∞–∑ (Create Order)

### Status: **FULLY FEASIBLE**

### Technical Implementation:

**Option A: Via Production API (cvety.kz)**
- ‚ùå **NOT AVAILABLE** - No order creation endpoint found in `/api/v2/`
- Production API only has:
  - `GET /products` - List products
  - `GET /orders` - List existing orders (admin only)

  - `POST /update-status` - Updates product status, not orders

**Option B: Via Railway Backend (Recommended)**
- ‚úÖ **AVAILABLE** - `POST /api/v1/orders/public/create`
- ‚úÖ **MCP Tool Available**: `create_order` (public, no auth required)

### Status: **FULLY FEASIBLE (Both Systems)**

### Option A: Via Production API

**Endpoint**: `GET /api/v2/orders?access_token=XXX`

**Capabilities**:
- ‚úÖ List all orders (requires admin token)
- ‚úÖ Filter by customer phone (need to check if supported)

## Use Case 3: ‚úÖ –ó–∞–≥—Ä—É–∑–∏—Ç—å —Ñ–æ—Ç–æ –±—É–∫–µ—Ç–∞ –∏ —É–∑–Ω–∞—Ç—å –ø–æ—Ö–æ–∂–∏–π (Visual Search)

### Status: **FULLY FEASIBLE (Railway Only)**

### Technical Implementation:

**System**: ‚úÖ Visual Search System (Cloudflare Worker + Vertex AI)

**MCP Tool**: `search_similar_bouquets(image_url, topK=5)`

**Architecture**:
```
Telegram Bot ‚Üí MCP Server ‚Üí Cloudflare Worker ‚Üí Google Vertex AI
                                ‚Üì
                          Vectorize Index (512D embeddings)
                                ‚Üì
                          PostgreSQL (product metadata)
```

**Capabilities**:
- ‚úÖ **Indexing**: 12 products indexed (10 real + 2 test)
- ‚úÖ **Accuracy**: 99.999% similarity for exact matches
- ‚úÖ **Speed**: 3.5 sec (warm cache), 7.4 sec (cold start)
- ‚úÖ **Similarity Thresholds**:
  - Exact match: >= 85% similarity
  - Similar: 70-85% similarity

**Telegram Bot Integration**:
```python

                f"{i+1}. {p['title']} - {p['price']} ‚Ç∏"
```

### Status: **PARTIALLY FEASIBLE (Railway Only)**

### Technical Implementation:

**Current Capabilities**:

- Cloudflare R2 bucket (`flower-shop-images`)
- Image upload endpoint: `https://flower-shop-images.alekenov.workers.dev/upload`

- Supported formats: PNG, JPEG, WebP

- `OrderPhoto` model exists in Railway backend

**Implementation Flow**:

1. **Florist uploads photo**:

**Challenges**:

‚ùå **Notification System Not Implemented**:
- No automatic Telegram notifications when photo is uploaded
- Requires webhook or polling mechanism
- Need to link orders to customer's Telegram chat_id

‚ùå **Florist Workflow Not Designed**:
- No UI for florist to upload pre-delivery photos
- Admin panel doesn't have photo upload for orders yet
- Telegram bot doesn't support florist photo uploads

‚ùå **Production API Limitation**:
- Production API (cvety.kz) has `mainImage` field in orders
- But no endpoint to upload/attach photos to existing orders
- Only read access to order images

**What Works Today**:
- ‚úÖ Railway backend can store order photos
- ‚úÖ Cloudflare R2 can host images
- ‚úÖ Telegram bot can send photos to customers
- ‚úÖ Testing endpoint exists for manual photo attachment

**What Needs to Be Built**:
- ‚ùå Admin UI for florist photo upload
- ‚ùå Notification webhook (order photo uploaded ‚Üí notify customer)
- ‚ùå Link orders to customer Telegram chat_id
- ‚ùå Approval workflow (customer confirms/rejects photo)

**Recommendation**: ‚ö†Ô∏è **FEASIBLE BUT REQUIRES DEVELOPMENT**

- No automatic notifications
